# Exercise 0.6: New Note in Single Page App Diagram

This diagram shows what happens when the user creates a new note in the **SPA** version (https://studies.cs.helsinki.fi/exampleapp/spa).

```mermaid
sequenceDiagram
    participant browser
    participant server

    Note right of browser: User types a note and clicks "Save"

    browser->>browser: JavaScript prevents form's default reload (preventDefault)
    browser->>browser: Adds new note to the notes list (re-renders instantly)
    browser->>server: POST https://studies.cs.helsinki.fi/exampleapp/new_note_spa
    activate server
    server-->>browser: 201 Created (success)
    deactivate server

    Note right of browser: Browser stays on the same page and shows updated notes
